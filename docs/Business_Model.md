# ビジネスモデル

## 収益モデル（案）

### オプションA：無料＋広告

| 項目 | 内容 |
|------|------|
| **基本方針** | アプリ本体は無料で提供 |
| **収益源** | 広告表示による収益 |
| **メリット** | ユーザー獲得のハードルが低い |
| **デメリット** | 広告が学習体験に影響する可能性 |

### オプションB：有料課金

| 項目 | 内容 |
|------|------|
| **基本方針** | 問題集の利用を有料化 |
| **収益源** | 月額課金 or 買い切り |
| **メリット** | 収益の予測がしやすい |
| **デメリット** | ユーザー獲得のハードルが高い |

### オプションC：ハイブリッド

| 項目 | 内容 |
|------|------|
| **基本方針** | 無料トライアル＋有料プレミアム |
| **収益源** | 広告（無料版）＋課金（プレミアム版） |
| **メリット** | 両方のメリットをバランス |
| **デメリット** | 設計・運用が複雑 |

※ モック段階では収益化は想定せず、本番開発時に決定

## コスト構造（想定）

| 項目 | 内容 |
|------|------|
| **開発コスト** | 人件費、外部委託費 |
| **運用コスト** | サーバー、ドメイン、保守 |
| **コンテンツコスト** | 問題作成・監修費用 |
| **マーケティング** | 認知拡大のための広告費 |

## 成功指標（KPI案）

| 指標 | 説明 |
|------|------|
| **DAU/MAU** | 日次・月次アクティブユーザー数 |
| **継続率** | リピート利用率 |
| **完了率** | 問題セットの完了率 |
| **収益** | 課金 or 広告収益（本番時） |

## スケーラビリティ

- 初期は小規模スタート
- ユーザー数・利用状況に応じて機能・インフラを拡張
- GMAP以外の試験への横展開の可能性
