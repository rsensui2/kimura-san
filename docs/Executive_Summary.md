# エグゼクティブサマリー

## ビジョン

> **「いつでも、どこでも、誰もが——。  
> スキマの5分が、未来を変える学びになる。」**

GMAP試験対策問題集アプリは、単なる問題集ではありません。通勤の電車、昼休みのカフェ、寝る前のベッド——どんな場所でも、スマホを開けばすぐに学べる。そんな「学びの民主化」を実現する第一歩です。

今日の小さな一問が、明日の合格につながる。そしてこのアプリが、GMAP受験者だけでなく、あらゆる学習者が自分らしいペースで成長できるプラットフォームへと進化していく——。そんな未来を、私たちは描いています。

## 2. 背景・課題認識

### GMAP試験とは

GMAPは、特定分野における知識・スキルを測定する試験であり、受験者数・認知度の拡大が期待されています。

### 市場・ユーザー環境の変化

- 資格・検定試験のデジタル化・オンライン学習の一般化
- スマートフォンによるスキマ時間学習の需要増加
- 紙媒体の教材からデジタル教材へのシフト

### 現状の課題

| 課題 | 詳細 |
|------|------|
| **学習手段の不足** | GMAP試験に特化した問題集・学習ツールが限られている |
| **利便性** | 紙の教材では、外出先やスキマ時間での学習が困難 |
| **フィードバック** | 即時採点・解説の提供が難しい |

### 提案の必然性

上記の課題を解決し、受験者の学習体験を向上させるため、デジタル問題集アプリの開発・提供を提案します。

## 3. 提案内容

### 提案の骨子

**GMAP試験対策問題集アプリ**の企画・開発を実施し、まずは**HTMLモック**を完成させることを提案します。

### アプリの構成案

#### (a) 動作環境

- **スマートフォン**：モバイルブラウザでの利用
- **ウェブ**：PCブラウザでの利用
- レスポンシブデザインにより、両環境で最適な表示を実現

#### (b) 最終ゴール

- **HTMLモックの作成**を第一フェーズの成果物とする
- モックにより、UI/UXの検証と関係者への説明・承認を実施
- 本番開発への移行判断の材料とする

### 開発アプローチ

1. **フェーズ1**：要件定義・設計
2. **フェーズ2**：HTMLモックの作成
3. **フェーズ3**：モックのレビュー・フィードバック反映
4. **フェーズ4**：本番開発の可否判断（別途検討）

### 想定機能（モック段階）

- 問題一覧表示
- 問題解答画面
- 採点・結果表示
- 解説表示（簡易版）
