# プロジェクトスケジュール

## 全体フェーズ

```
フェーズ1（要件・設計） → フェーズ2（モック作成） → フェーズ3（レビュー） → 判断
```

## フェーズ1：要件定義・設計

| タスク | 内容 | 目安期間 |
|--------|------|----------|
| 要件整理 | 機能一覧、優先度の確定 | 1週間 |
| ワイヤーフレーム | 画面遷移・レイアウトの設計 | 1週間 |
| デザイン方針 | カラー、フォント、UIガイドライン | 数日 |

## フェーズ2：HTMLモック作成

| タスク | 内容 | 目安期間 |
|--------|------|----------|
| 基盤構築 | HTML/CSSのテンプレート作成 | 1週間 |
| 画面実装 | 各画面の実装 | 2〜3週間 |
| サンプルデータ | 問題データの準備 | 並行 |
| 動作確認 | クロスブラウザ・レスポンシブ確認 | 数日 |

## フェーズ3：レビュー・フィードバック

| タスク | 内容 | 目安期間 |
|--------|------|----------|
| 社内レビュー | 関係者によるモック確認 | 1週間 |
| フィードバック反映 | 指摘事項の修正 | 1週間 |
| 最終確認 | 承認・本番開発判断 | 数日 |

## マイルストーン

| マイルストーン | 内容 |
|----------------|------|
| **M1** | 要件定義完了・設計承認 |
| **M2** | HTMLモック初版完成 |
| **M3** | モックレビュー完了 |
| **M4** | 本番開発の可否判断 |

## 注意事項

- 上記は目安であり、リソース・優先度に応じて調整
- GMAP試験の公式情報・コンテンツ利用については、権利関係の確認が必要
